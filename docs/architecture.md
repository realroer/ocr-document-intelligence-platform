# OCR Document Intelligence Platform Architecture

## Overview

The platform transforms OCR output generated by Google Document AI into a standardized payroll dataset suitable for downstream analytics, reporting, and ETL workflows.

---

## Processing Flow

```
Incoming Documents
        │
        ▼
Google Document AI
        │
        ▼
OCR JSON
        │
        ▼
Batch Processor
        │
        ▼
Layout Detection
        │
        ▼
Field Extraction
        │
        ├──────────────┐
        ▼              ▼
 Payments        Deductions
        │              │
        └──────┬───────┘
               ▼
Normalization
               ▼
Validation
               ▼
Duplicate Detection
               ▼
JSON / CSV Export
               ▼
Master Dataset
```

---

## Main Components

### Batch Processor

Coordinates document processing and manages batch execution.

### Layout Detection

Automatically identifies document templates from multiple payroll providers.

### Field Extraction

Extracts structured payroll information including:

- Employee Information
- Employer Information
- Salary Payments
- Mandatory Deductions
- Voluntary Deductions
- Pension Contributions
- Vacation Accumulators
- Sick Leave Accumulators
- Summary Fields

### Normalization

Converts extracted values into a unified internal schema independent of the original document layout.

### Export

Produces standardized JSON and CSV outputs suitable for analytics pipelines.

---

## Design Principles

- Modular architecture
- Layout-independent extraction
- Reusable parsing components
- Easy integration of new payroll layouts
- Analytics-ready output
