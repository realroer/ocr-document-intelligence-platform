# Project Story

## Background

Payroll documents were received from multiple employers, each using different layouts and formatting. Although Google Document AI provided OCR results, the extracted information required significant normalization before it could be used for reporting or analytics.

## Challenge

Different document layouts contained payroll information in different positions and structures. A single extraction strategy was not sufficient to process all documents accurately.

## Solution

Developed a modular Python-based OCR processing platform that:

- Processes OCR JSON generated by Google Document AI
- Detects document layouts automatically
- Applies layout-aware extraction logic
- Normalizes payroll information into a unified schema
- Removes duplicate documents
- Produces analytics-ready JSON and CSV datasets

## Results

- Automated multi-document processing
- Support for multiple payroll layouts
- Consistent structured output
- Modular extraction architecture
- Foundation for future API and machine learning enhancements

## Lessons Learned

This project strengthened practical experience in:

- OCR post-processing
- Document Intelligence
- ETL pipeline design
- Data normalization
- Modular Python architecture
- Production-ready data engineering
